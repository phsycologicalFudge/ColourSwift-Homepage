Overview

CS Security is a modern Android antivirus and behavioural scanner that combines trusted open-source intelligence with custom proprietary systems. All analysis occurs entirely on your device, ensuring no file content or metadata ever leaves your phone.

Our encrypted VXPack includes local signature databases derived from ClamAV’s trusted virus definitions, alongside our own dataset from CS Security’s AV Database. This combination allows for strong hybrid detection while preserving your privacy.

Core Design

  • Privacy First: 100% on-device detection, no cloud scanning or data upload.
  • Hybrid Intelligence: Uses ClamAV signatures and local ML-based heuristics together.
  • Lightweight Performance: Native Rust engine optimized for speed and low memory.
  • Transparency: Public technical documentation for users and researchers.

ClamAV Integration

CS Security integrates with ClamAV, a respected open-source antivirus engine. ClamAV definitions form the foundation of our VXPack, giving CS Security access to thousands of verified threat signatures without requiring an online connection.

How It’s Used

  • CS Security periodically fetches updated ClamAV signature sets and compresses them into its VXPack format.
  • Definitions are AES-encrypted locally and verified before loading to ensure authenticity.
  • During scans, CS Security cross-references file hashes and byte sequences against both ClamAV’s and CS Security’s internal threat lists.

Open Collaboration

To maintain transparency and extend malware coverage, we provide our own public AV Database repository containing definitions and releases of our file manager. Developers or analysts can learn more about ClamAV’s format and logic at docs.clamav.net.

How It Works

The CS Security engine processes files through several carefully controlled steps. While implementation details are proprietary, the workflow below illustrates how scanning and threat detection occur.

1. Engine Initialization

  • The encrypted VXPack (containing ClamAV and CS definitions) is loaded and decrypted locally.
  • All signature tables, hash maps, and ML models are verified and prepared in memory.
  • The engine allocates minimal resources to maintain smooth performance during scanning.

2. Scan Modes

  • Smart Scan: Balanced speed and accuracy, analysing essential system and app directories.
  • Rapid Scan: Fastest mode for quick surface-level checks.
  • Single File Scan: Manual scan for any selected file or APK.

3. Detection Layers

  1. Signature Layer: Compares file patterns against ClamAV and CS definitions.
  2. Hash Layer: Uses SHA-256 fingerprints to confirm known malware samples.
  3. Heuristic Layer: Applies AI-assisted pattern analysis and APK behaviour scoring.

Once analysis completes, CS Security provides a clear verdict. Clean, Suspicious, or Malicious, displayed in the app UI.

Privacy & Safety

CS Security never sends any scanned data, hashes, or results to external servers. The scanner operates fully offline and isolates your system files from external access.

  • No analytics SDKs or third-party tracking libraries.
  • All updates verified via cryptographic checksum before installation.
  • Only technical metadata is read during scanning, never personal content.

This ensures that your device remains protected without compromising your privacy, an approach inspired by open-source values but applied through proprietary safeguards.

Machine Learning

CS Security uses an offline-trained behavioural model that analyses APKs beyond static signatures. The model inspects app structures, permission usage, and declared components to estimate potential risk.

  • Trained on both benign and malicious APK samples.
  • Focuses on statistical patterns, not user data or device telemetry.
  • Runs entirely on-device with no remote inference.

This layer assists in identifying new or obfuscated threats that may bypass traditional pattern-based checks, helping CS Security stay adaptive while preserving privacy.

Limitations

No security product is perfect. CS Security is designed for practicality, speed, and privacy, and is a newborn in the mist of giant corporations.

  • Highly encrypted or self-mutating payloads may avoid detection.
  • Threats exploiting kernel or firmware vulnerabilities fall outside app-level scope.
  • False positives can occur with new or heavily obfuscated software.

Best Practices

  • Keep your VXPack definitions updated regularly.
  • Use Smart Scan weekly, Rapid Scan for quick checks, and Single File Scan for manual testing.
  • Install apps only from trusted sources such as Google Play or verified repositories.

CS Security’s hybrid model, ClamAV's definitions plus proprietary layers, ensures powerful offline protection while remaining lightweight, transparent, and respectful of user trust.
